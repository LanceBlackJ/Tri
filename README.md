## 1. 系统概述

Tri是一款基于AI技术的个性化学习教育系统，旨在为学生提供量身定制的学习体验。

系统通过用户与AI交互为用户量身定制多维度学习画像，并可以基于用户画像为用户生成定制化的学习方案。

### 主要功能

| 功能模块 | 功能描述 |
| :--- | :--- |
| **学习画像** | 通过对话式交互构建6维度的学习画像，了解学生的学习偏好和能力水平 |
| **课程生成** | 基于学习画像自动生成课程大纲、教学课件和练习题 |
| **AI辅导** | 苏格拉底式问答和多模态解答，提供个性化的学习指导 |
| **进度追踪** | 实时跟踪学习进度，评估学习效果 |
| **知识追踪** | 评估学生对各个知识点的掌握程度，识别薄弱环节 |
| **学习路径** | 根据学习画像和知识状态，智能规划最优学习路径 |
| **情感识别** | 分析学生的学习情绪，提供针对性的激励和支持 |
| **参与度分析** | 量化分析学生的学习参与度，优化学习体验 |
| **学习评估** | 提供多种评估方式，全面检测学习成果 |

### 适用场景

- **课堂教学**：教师使用系统生成课件，支持课堂讲解
- **自主学习**：学生通过系统进行个性化学习和知识巩固
- **在线辅导**：结合AI对话功能，提供一对一的学习辅导

## 2. 安装与配置

### 2.1 环境要求

- Python 3.10+
- Django 4.2+
- Docker & Docker Compose
- 讯飞星火大模型 API 凭证

### 2.2 安装步骤

#### 步骤1：克隆项目

```bash
git clone https://github.com/LanceBlackJ/GimmeClass
cd teacher_django
```

#### 步骤2：创建虚拟环境

**Windows**：
```powershell
python -m venv venv
venv\Scripts\activate
```

**Linux/Mac**：
```bash
python -m venv venv
source venv/bin/activate
```

#### 步骤3：安装依赖

```bash
pip install -r requirements.txt
```

#### 步骤4：配置环境变量

复制环境变量模板并编辑：

```bash
cp .env.example .env
```

编辑 `.env` 文件，填入以下必填项：

```env
# 讯飞星火 API 凭证（必须）
XUNFEI_APPID=your_appid_here
XUNFEI_API_KEY=your_api_key_here
XUNFEI_API_SECRET=your_api_secret_here

# 应用密钥（生产环境必须修改）
SECRET_KEY=your_secret_key_here

# 调试模式（生产环境设为 False）
DEBUG=True

# 允许的主机（生产环境添加域名）
ALLOWED_HOSTS=127.0.0.1,localhost
```

#### 步骤5：数据库迁移

```bash
python manage.py makemigrations
python manage.py migrate
```

#### 步骤6：创建管理员账户（可选）

```bash
python manage.py createsuperuser
```

### 2.3 启动服务

#### 开发环境

```bash
python manage.py runserver
```

访问 `http://127.0.0.1:8000` 即可使用系统。

#### 生产环境

生产环境建议使用 Docker 部署 Redis 和 Celery，以处理后台任务（如课程生成、PPT导出等耗时操作）。

##### Docker 部署 Redis 和 Celery

**步骤1：安装 Docker**

确保系统已安装 Docker 和 Docker Compose：

- Windows/Mac：下载安装 [Docker Desktop](https://www.docker.com/products/docker-desktop)
- Linux：参考 [Docker 官方文档](https://docs.docker.com/engine/install/) 安装

**步骤2：配置 Docker Compose**

项目根目录下已提供 `docker-compose.yml` 配置文件，包含 Redis 和 Celery 服务：

```yaml
version: '3.8'

services:
  redis:
    image: redis:7-alpine
    container_name: teacher_redis
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  celery_worker:
    build:
      context: .
      dockerfile: Dockerfile.celery
    container_name: teacher_celery_worker
    depends_on:
      redis:
        condition: service_healthy
    environment:
      - CELERY_BROKER_URL=redis://redis:6379/0
      - CELERY_RESULT_BACKEND=redis://redis:6379/0
    volumes:
      - .:/app
    restart: unless-stopped

volumes:
  redis_data:
```

**步骤3：启动 Docker 服务**

在项目根目录执行：

```bash
docker-compose up -d
```

此命令会：
- 启动 Redis 容器（端口 6379）
- 启动 Celery Worker 容器
- 自动建立服务间的连接

**步骤4：验证服务状态**

检查容器运行状态：

```bash
docker-compose ps
```

预期输出：

```
NAME                  STATUS    PORTS
teacher_redis         running   0.0.0.0:6379->6379/tcp
teacher_celery_worker running
```

检查 Redis 连接：

```bash
docker exec -it teacher_redis redis-cli ping
```

预期返回：`PONG`

**步骤5：查看 Celery 日志**

查看 Celery Worker 运行日志：

```bash
docker-compose logs -f celery_worker
```

##### Docker 服务管理命令

| 命令 | 描述 |
| :--- | :--- |
| `docker-compose up -d` | 启动所有服务（后台运行） |
| `docker-compose down` | 停止并删除所有容器 |
| `docker-compose restart` | 重启所有服务 |
| `docker-compose ps` | 查看服务运行状态 |
| `docker-compose logs -f` | 查看实时日志 |
| `docker-compose logs celery_worker` | 查看 Celery 日志 |
| `docker-compose logs redis` | 查看 Redis 日志 |

##### 配置 Django 连接 Docker 服务

在 `.env` 文件中添加以下配置：

```env
CELERY_BROKER_URL=redis://localhost:6379/0
CELERY_RESULT_BACKEND=redis://localhost:6379/0
```

##### 手动启动方式（备选）

如不使用 Docker，可手动启动服务：

**启动 Redis**：

```powershell
.\scripts\start_redis.ps1
```

**启动 Celery Worker**：

```powershell
.\scripts\start_celery_worker.ps1
```

**启动开发服务器**：

```powershell
.\scripts\start_dev.ps1
```

## 5. 学习画像构建

学习画像系统通过AI对话式交互，从多个维度了解学生的学习情况，为后续的个性化课程生成和学习路径规划提供依据。本系统基于多维度学生模型构建方法 [8] 和认知风格理论 [9]，能够全面捕捉学生的学习特征。

### 5.1 画像维度

系统构建的学习画像包含以下六个维度 [8]：

| 维度 | 描述 | 示例 |
| :--- | :--- | :--- |
| **知识基础** | 已掌握的知识点与程度 | 基础薄弱、中等水平、"熟练掌握 |
| **认知风格** | 视觉/听觉/动手等学习偏好 | 视觉型、听觉型、实践型 |
| **学习节奏** | 快慢节奏与学习习惯 | 快速学习、循序渐进 |
| **元认知** | 自我反思与评估能力 | 善于总结、需要引导反思 |
| **学习动机** | 内在/外在驱动因素 | 兴趣驱动、目标导向 |
| **易错模式** | 常见错误与薄弱点 | 概念混淆、计算错误 |

### 5.2 构建流程

#### 步骤1：进入画像构建页面

点击顶部导航栏的"学习画像"，进入学习画像构建页面。

#### 步骤2：查看画像维度说明

页面顶部展示六个画像维度的说明卡片，帮助您了解每个维度的含义：
- 知识基础：已掌握的知识点与程度
- 认知风格：视觉/听觉/动手等偏好
- 学习节奏：快慢节奏与学习习惯
- 元认知：自我反思与评估能力
- 学习动机：内在/外在驱动因素
- 易错模式：常见错误与薄弱点

#### 步骤3：AI对话引导

在左侧对话面板中，AI会通过一系列引导性问题了解您的学习情况：
- 系统会根据您的回答自动分析并更新画像
- 您可以随时跳过某个维度的问题
- 对话进度会实时显示在进度条中

#### 步骤4：查看画像预览

右侧预览面板会实时显示当前构建的画像内容：
- 已完成的维度会标记为"已完成"
- 画像摘要包含各维度的分析结果
- 可以查看具体的画像数据

#### 步骤5：完成构建

当所有维度都完成或跳过后，点击"完成"按钮保存学习画像。

### 5.3 更新画像

学习画像并非一成不变，您可以随时更新：

1. 进入"学习画像"页面
2. 点击"重新生成画像"按钮
3. 重新进行AI对话引导
4. 系统会根据新的对话内容更新画像

### 5.4 画像应用

构建完成的学习画像会应用于：
- **课程生成**：根据认知风格和学习节奏生成适合的课程内容
- **学习路径规划**：根据知识基础和易错模式规划最优学习路径
- **AI辅导**：根据学习动机和元认知能力调整辅导策略

### 5.5 实现方式

学习画像采用多数据源融合的方式构建，系统从多个来源获取和分析数据，综合评估学生的学习特征。

#### 数据来源

系统从以下三个维度收集画像构建所需的数据：

| 数据源 | 描述 | 获取方式 |
| :--- | :--- | :--- |
| **用户对话** | 通过AI对话引导收集用户的反馈信息 | AI对话系统实时分析 |
| **历史数据** | 记录用户的学习历史和表现 | 学习记录数据库 |
| **知识评估** | 通过测验评估用户的知识掌握情况 | 练习题答题结果 |

#### 画像维度量化

系统将5.1节的六个画像维度量化为雷达图展示数据，具体映射关系如下：

| 画像维度（5.1节） | 雷达图展示维度 | 量化方式 | 满分阈值 |
| :--- | :--- | :--- | :--- |
| **知识基础** | **知识掌握** | 知识点得分的平均值（0-100分） | 100 |
| **认知风格、学习节奏** | **学习偏好** | 已填写的偏好项比例 | 100 |
| **元认知、易错模式** | **概念清晰度** | 根据错误概念数量计算，错误越少得分越高 | 100 |
| **学习动机** | **学习目标** | 根据学习目标数量计算 | 100 |
| 行为数据综合 | **学习参与度** | 根据学习行为综合计算 | 100 |
| 系统评估 | **画像置信度** | 根据数据可靠性评估 | 100 |

#### 画像数据结构

系统构建的学习画像包含以下核心数据：

| 数据类别 | 描述 |
| :--- | :--- |
| **知识掌握情况** | 各知识点的掌握程度评估 |
| **认知风格** | 学生的学习风格描述 |
| **学习目标** | 学生设定的学习目标列表 |
| **错误概念** | 已识别的错误理解和易错点 |
| **参与度数据** | 学习行为的量化评估 |
| **学习偏好** | 偏好的学习方式和节奏 |
| **置信度评估** | 各维度数据的可靠性评分 |

#### 画像构建流程

学习画像的构建遵循以下步骤：

| 步骤 | 描述 |
| :--- | :--- |
| **1. 会话初始化** | 创建画像构建会话，准备收集数据 |
| **2. 问题生成** | 根据当前画像状态生成引导性问题 |
| **3. 答案收集** | 记录用户回答并分析其含义 |
| **4. 画像更新** | 将分析结果整合到学习画像中 |
| **5. 成长报告** | 生成画像变化趋势报告，展示进步情况 |

## 6. 课程生成

系统基于学习画像自动生成个性化课程内容，包括课程大纲、教学课件、动画演示、代码示例和练习题。

### 6.1 课程生成流程

#### 步骤1：进入课程生成页面

点击顶部导航栏的“AI课件”，进入智能课程生成器页面。

#### 步骤2：输入学习主题

在"学习主题"输入框中输入您想学习的内容，例如：
- 计算机组成原理中的总线
- 机器学习里的梯度下降
- 线性代数基础

#### 步骤3：补充学习目标（可选）

在“学习目标与基础”文本框中，可以补充以下信息：
- 您的基础水平（如“我没有基础”、“有一定编程经验”）
- 期望的学习时长（如“希望用20分钟听懂概念”）
- 特殊需求（如“配一个动态演示”、“包含代码示例”）

#### 步骤4：选择生成选项

系统提供以下生成选项，默认全部勾选：

| 选项 | 描述 |
| :--- | :--- |
| **生成练习题** | 为课程生成配套的测验题目 |
| **包含动画演示** | 生成交互式动画来演示概念 |
| **代码示例** | 为编程相关课程生成代码示例 |

#### 步骤5：开始生成

点击"开始生成课程"按钮，系统会根据您的输入和选项生成课程。生成过程可能需要几分钟，页面会实时显示生成进度。

### 6.2 课程内容

生成的课程包含以下内容：

| 内容类型 | 描述 |
| :--- | :--- |
| **课程大纲** | 结构化的课程章节安排，包含多个阶段 |
| **教学课件** | 网页形式的交互式课件，支持PPT风格播放 |
| **动画演示** | 可视化的概念动画，帮助理解抽象概念 |
| **练习题** | 配套的测验题目，支持即时反馈 |
| **代码示例** | 编程相关课程的代码示例和解释 |

### 6.3 课程学习

#### 进入课程学习页

课程生成完成后，点击课程卡片进入课程详情页面，可以：
- 查看课程大纲结构
- 播放PPT课件
- 观看动画演示
- 完成练习题
- 查看代码示例

#### 课程大纲学习

课程详情页面展示完整的课程大纲，包含多个阶段：
- 每个阶段包含多个学习节点
- 点击节点可以查看详细内容
- 学习进度会实时更新

### 6.4 课程管理

#### 查看课程列表

点击顶部导航栏的“学习中心”，查看所有已生成的课程，包含：
- 课程标题和描述
- 生成状态
- 预计学习时长
- 创建时间

#### 删除课程

在课程列表页面，勾选需要删除的课程，点击"批量删除"按钮，确认后删除课程。

### 6.5 实现方式

课程生成系统采用基于学习画像的智能生成方式，通过多个步骤实现个性化课程内容的创建。

#### 核心流程

## 10. 知识追踪

知识追踪功能通过分析学生的学习行为和测验成绩，评估学生对各个知识点的掌握程度，帮助识别学习薄弱环节。本系统采用深度知识追踪（Deep Knowledge Tracing, DKT）[1] 和贝叶斯知识追踪（Bayesian Knowledge Tracing, BKT）[2] 相结合的方法，实现精准的知识状态评估。

### 10.1 知识状态可视化

系统以图表形式展示知识掌握情况 [1]：

| 可视化方式 | 描述 |
| :--- | :--- |
| **知识图谱** | 以网络图形式展示知识点之间的关联和掌握程度 |
| **热力图** | 用颜色深浅表示不同知识点的掌握水平 |
| **进度条** | 直观展示每个知识点的学习进度 |

### 10.2 知识点掌握程度

系统将知识点掌握程度分为五个等级 [1]：

| 等级 | 描述 | 颜色标识 | 判断依据 |
| :--- | :--- | :--- | :--- |
| **未学习** | 尚未开始学习该知识点 | 灰色 | wrong_count等于0 |
| **入门** | 初步了解基本概念，存在少量错误 | 蓝色 | wrong_count大于0且consecutive_wrong_count为0或1 |
| **理解** | 存在理解问题，连续错误较多 | 绿色 | wrong_count大于0且consecutive_wrong_count大于1 |
| **熟练** | 正在进步，错误减少，正确次数增加 | 黄色 | wrong_count大于0但recent_corrects累计增加 |
| **精通** | 稳定掌握，无错误或持续正确 | 红色 | wrong_count等于0且recent_corrects持续增加 |

### 10.3 薄弱环节识别

系统会自动识别学习薄弱环节，并提供改进建议 [1]：

1. **智能检测**：通过测验成绩和学习行为数据分析薄弱环节
2. **优先级排序**：根据知识点的重要性和掌握程度排序
3. **学习建议**：针对薄弱环节提供具体的学习建议和资源推荐

### 10.4 知识追踪报告

系统定期生成知识追踪报告：

- **知识点掌握概览**：各知识点的掌握程度统计
- **进步趋势**：一段时间内的知识掌握变化趋势
- **薄弱环节清单**：需要重点加强的知识点列表
- **学习建议**：基于知识状态的个性化学习建议

### 10.5 知识追踪算法

系统采用两种知识追踪算法的融合 [1][2]：

| 算法 | 特点 | 应用场景 |
| :--- | :--- | :--- |
| **深度知识追踪 (DKT)** | 使用深度学习模型，能够捕捉复杂的知识状态转换 | 大规模数据、复杂知识结构 |
| **贝叶斯知识追踪 (BKT)** | 基于贝叶斯网络，模型可解释性强 | 小规模数据、简单知识结构 |

### 10.6 实现方式

知识追踪系统通过分析测验答题数据来评估知识掌握程度。

#### 数据收集与分析

系统收集和分析以下类型的数据：

| 数据类型 | 收集方式 |
| :--- | :--- |
| **答题记录** | 用户完成练习后记录答题结果 |
| **知识点标记** | 记录每道题对应的知识点标签 |
| **错误统计** | 统计每个知识点的错误次数 |
| **连续错误** | 追踪连续错误次数 |
| **最后出现** | 记录知识点最后出现时间 |
| **来源追踪** | 记录错误的来源页面和标题 |

#### 知识状态评估

系统根据答题情况评估知识掌握状态：

| 评估指标 | 计算方式 | 状态判断 |
| :--- | :--- | :--- |
| **未学习** | 错误次数为零 | 尚未开始学习 |
| **入门** | 存在少量错误，连续错误较少 | 初步了解 |
| **理解** | 存在错误且连续错误较多 | 存在理解问题 |
| **熟练** | 存在错误但正确次数在增加 | 正在进步 |
| **精通** | 无错误或持续正确 | 稳定掌握 |

#### 薄弱环节识别

系统通过以下方式识别薄弱环节：

| 识别方式 | 描述 |
| :--- | :--- |
| **错误次数排序** | 按错误次数从多到少排列 |
| **连续错误排序** | 按连续错误次数从多到少排列，优先处理 |
| **最近出现排序** | 按最近出现时间排序 |
| **归档排除** | 排除已阶段性处理完成的薄弱点 |

## 11. 学习路径规划

学习路径规划功能根据学生的学习画像和当前知识状态，智能规划最优的学习路径。

### 11.1 路径生成算法

系统综合考虑以下因素生成学习路径：

| 因素 | 描述 |
| :--- | :--- |
| **学习目标** | 学生的学习目标和期望 |
| **知识状态** | 当前对各知识点的掌握程度 |
| **知识依赖** | 知识点之间的先后顺序和依赖关系 |
| **学习风格** | 偏好的学习方式和节奏 |
| **可用时间** | 可用于学习的时间安排 |

### 11.2 路径类型

系统支持多种学习路径类型：

| 路径类型 | 描述 | 适用场景 |
| :--- | :--- | :--- |
| **循序渐进** | 按照知识体系的逻辑顺序学习 | 初学者、基础薄弱者 |
| **强化训练** | 针对薄弱环节进行集中训练 | 需要提升特定技能者 |
| **快速突破** | 跳过已掌握内容，直接学习新知识 | 基础较好者 |
| **综合提升** | 全面覆盖所有知识点，均衡发展 | 备考、全面复习 |

### 11.3 路径调整

学习路径并非一成不变，可以根据学习情况调整：

1. **自动调整**：系统根据学习进度和成绩自动优化路径
2. **手动调整**：学生可以手动调整学习顺序和重点
3. **教师指导**：教师可以为学生定制学习路径

### 11.4 路径执行

按照规划的学习路径进行学习：

- **每日任务**：系统根据路径生成每日学习任务
- **进度跟踪**：实时跟踪学习路径的执行进度
- **完成度统计**：统计各阶段任务的完成情况
- **里程碑提醒**：在重要节点提供提醒和鼓励

### 11.5 实现方式

学习路径规划系统采用多因素融合的路径生成方式，综合考虑学习画像、知识状态和学习目标等因素。

#### 路径生成流程

学习路径生成遵循以下步骤：

| 步骤 | 描述 |
| :--- | :--- |
| **1. 主题规范化** | 提取和规范化学习主题 |
| **2. 画像加载** | 加载用户学习画像 |
| **3. 课程匹配** | 匹配相关的已有课程 |
| **4. 薄弱点识别** | 查询最近的错误知识点 |
| **5. 三阶段规划** | 生成三个阶段的学习任务 |

#### 路径数据结构

系统生成的学习路径包含三个阶段：

| 阶段 | 名称 | 学习任务数 | 预计时长 | 重点 |
| :--- | :--- | :--- | :--- | :--- |
| **阶段一** | 先搭框架再进入细节 | 2个任务 | 3.5小时 | 建立全局认知 |
| **阶段二** | 结合案例和练习完成理解迁移 | 2个任务 | 4.0小时 | 概念落地和薄弱点补强 |
| **阶段三** | 通过复盘形成稳定掌握 | 1个任务 | 2.0小时 | 综合练习和错因分析 |

#### 每个阶段的学习任务结构

每个学习任务包含以下内容：

| 内容类别 | 描述 | 示例值 |
| :--- | :--- | :--- |
| **任务标题** | 任务的名称 | "先建立梯度下降的整体框架" |
| **任务目标** | 任务要达到的目标 | "先回答核心概念有哪些" |
| **资源类型** | 需要的学习资源类型 | ["文档", "PPT"] |

#### 调优触发机制

系统支持多种路径调优触发方式：

| 触发来源 | 描述 | 处理方式 |
| :--- | :--- | :--- |
| **资料小测** | 通过练习答题触发 | 根据答题结果调整学习路径 |
| **手动重排** | 用户主动触发 | 用户请求重新规划路径 |
| **薄弱点重排** | 薄弱点更新触发 | 优先安排薄弱点学习 |

#### 动态调整机制

| 调整方式 | 描述 | 更新字段 |
| :--- | :--- | :--- |
| **课程关联** | 关联匹配的课程资料 | matched_course字段 |
| **阶段重点** | 更新当前阶段的重点 | top_module_name和top_module_focus |
| **薄弱点列表** | 更新需要补强的知识点 | weak_areas字段 |
| **推荐理由** | 生成推荐原因说明 | recommendation_reason字段 |

## 12. 情感识别

情感识别功能通过分析学生的对话内容和学习行为，识别学生的学习情绪状态，提供针对性的激励和支持。本系统基于OCC情感模型 [3] 和并行同理心与反应性同理心理论 [4]，实现精准的情感识别和响应。

### 12.1 情感维度

系统识别以下情感维度 [3]：

| 情感类型 | 描述 | 表现 |
| :--- | :--- | :--- |
| **积极** | 学习状态良好，情绪高涨 | 主动提问、积极互动、按时完成任务 |
| **中性** | 学习状态平稳，情绪正常 | 按部就班学习，没有明显波动 |
| **消极** | 学习状态不佳，情绪低落 | 频繁出错、拖延任务、消极反馈 |
| **焦虑** | 学习压力大，感到焦虑 | 频繁寻求帮助、担心考试、情绪紧张 |
| **困惑** | 对内容理解困难，感到困惑 | 反复提问、请求解释、进度缓慢 |

### 12.2 识别方法

系统通过多种方式识别情感状态 [3]\[4]：

| 识别方式 | 描述 |
| :--- | :--- |
| **文本分析** | 分析对话内容中的情感词汇和语气 |
| **行为分析** | 根据学习行为模式判断情绪状态 |
| **进度分析** | 通过学习进度变化推断学习状态 |
| **反馈分析** | 分析学生对学习内容的反馈和评价 |

### 12.3 情感干预

当系统检测到消极情绪时，会采取以下干预措施 [4]：

| 干预方式 | 描述 |
| :--- | :--- |
| **鼓励激励** | 提供积极的鼓励和肯定 |
| **学习建议** | 根据情绪状态调整学习内容和难度 |
| **休息提醒** | 建议适当休息，调整状态 |
| **问题解答** | 针对困惑提供详细解答 |
| **心理支持** | 提供学习心理调适建议 |

### 12.4 情感报告

系统生成情感分析报告：

- **情感状态趋势**：一段时间内的情绪变化趋势
- **情绪触发因素**：导致情绪波动的主要因素
- **干预效果评估**：干预措施的效果评估
- **建议改进措施**：改善学习情绪的建议

### 12.5 同理心机制

系统采用并行同理心与反应性同理心机制 [4]：

| 同理心类型 | 描述 | 应用场景 |
| :--- | :--- | :--- |
| **并行同理心** | 系统主动感知学生情感状态，提前预判需求 | 日常学习过程中 |
| **反应性同理心** | 针对学生情感变化做出及时响应和支持 | 学生遇到困难或情绪波动时 |

### 12.6 实现方式

情感识别系统通过分析对话内容和学习行为来识别情感状态，综合多个数据源进行评估。

#### 情感分析数据来源

系统从多个数据源综合分析情感状态：

| 数据类型 | 分析方式 |
| :--- | :--- |
| **对话事件记录** | 记录用户进入AI辅导、资源推送等行为事件 |
| **对话内容分析** | 分析AI对话中的情感词汇和语气 |
| **学习行为数据** | 分析学习频率、时长等行为数据 |
| **参与度得分** | 参与度的量化评估值 |
| **测验反馈数据** | 分析练习答题时的反馈和情绪 |

#### 情感状态判断

系统根据多个指标综合判断情感状态：

| 情感状态 | 判断依据 |
| :--- | :--- |
| **积极** | 参与度得分较高且稳定 |
| **中性** | 正常学习，没有明显波动 |
| **消极** | 参与度得分较低或错误次数持续增加 |
| **焦虑** | 频繁寻求帮助、对话中出现焦虑词汇 |
| **困惑** | 反复提问、请求解释 |

#### 事件记录机制

| 事件类型 | 描述 | 置信度 |
| :--- | :--- | :--- |
| **资源推送访问** | 资源推送页面被访问 | 0.2 |
| **AI辅导访问** | AI辅导页面被访问 | 0.45 |
| **画像构建步骤** | 画像构建步骤完成 | 0.3-0.6 |
| **学习进度更新** | 学习进度更新 | 0.25-0.35 |

#### 并行同理心实现

系统主动感知学生情感状态，在以下场景提前预判需求：

| 场景 | 预判方式 | 响应策略 |
| :--- | :--- | :--- |
| **进入AI辅导** | 记录AI辅导访问事件 | 准备辅导内容和建议问题 |
| **进入资源推送** | 记录资源推送访问事件 | 分析推荐资源相关性 |
| **画像构建** | 记录画像构建步骤事件 | 调整引导问题策略 |

#### 反应性同理心实现

针对学生情感变化做出及时响应：

| 响应场景 | 检测方式 | 响应内容 |
| :--- | :--- | :--- |
| **困惑检测** | 检测到困惑相关的提问 | 提供针对性的学习建议 |
| **薄弱点检测** | 识别到知识薄弱点 | 生成补弱建议 |
| **学习焦点** | 分析当前学习状态 | 生成当前学习焦点分析 |

## 13. 参与度分析

参与度分析功能量化分析学生的学习参与度，帮助了解学生的学习投入情况。本系统基于三维度参与度模型 [5]，从认知、情感和行为三个维度全面评估学习参与度。

### 13.1 参与度指标

系统从多个维度评估参与度 [5]：

| 指标 | 描述 | 计算方式 |
| :--- | :--- | :--- |
| **学习时长** | 投入学习的总时间 | 累计学习时长 |
| **互动频率** | 与AI辅导的互动次数 | 对话次数、提问次数 |
| **任务完成率** | 完成学习任务的比例 | 已完成任务/总任务数 |
| **测验参与率** | 参与测验的比例 | 已完成测验/总测验数 |
| **内容浏览** | 浏览学习内容的深度 | 浏览页面数、阅读时长 |
| **参与度得分** | 综合量化得分 | engagement.score字段值（0-100） |

### 13.2 参与度等级

系统将参与度分为四个等级：

| 等级 | 描述 | 特征 |
| :--- | :--- | :--- |
| **高参与** | 积极主动学习 | 学习时长充足、互动频繁、任务完成率高 |
| **中参与** | 正常学习状态 | 按时完成任务，适度互动 |
| **低参与** | 学习积极性不足 | 学习时长较短、任务完成率低 |
| **不参与** | 长期未学习 | 连续多日未登录或未学习 |

### 13.3 参与度提升

系统提供以下方式提升学习参与度：

| 提升方式 | 描述 |
| :--- | :--- |
| **目标设定** | 帮助设定合理的学习目标 |
| **进度激励** | 在关键节点提供激励和奖励 |
| **内容优化** | 根据兴趣推荐个性化内容 |
| **互动设计** | 设计有趣的互动环节 |
| **社交功能** | 提供学习社区和同伴互动 |

### 13.4 参与度报告

系统生成参与度分析报告：

- **参与度概览**：各项参与度指标的统计
- **参与度趋势**：一段时间内的参与度变化趋势
- **对比分析**：与平均水平的对比分析
- **提升建议**：针对低参与度的改进建议

### 13.5 三维度参与度模型

系统采用三维度参与度模型 [5]：

| 维度 | 描述 | 评估方式 |
| :--- | :--- | :--- |
| **认知参与度** | 衡量学生的注意力、思考深度和认知投入 | 学习深度、问题解决能力、知识应用 |
| **情感参与度** | 衡量学生的学习兴趣、动机和情绪状态 | 学习态度、情绪变化、自我效能感 |
| **行为参与度** | 衡量学生的学习行为、互动频率和任务完成情况 | 学习时长、互动次数、任务完成率 |

### 13.6 实现方式

参与度分析系统通过量化学习行为数据来评估参与度，综合多个维度进行评估。

#### 参与度数据收集

系统收集以下类型的数据：

| 数据类型 | 收集方式 |
| :--- | :--- |
| **学习时长** | 记录用户学习时的累计时长 |
| **互动频率** | 统计AI对话和提问次数 |
| **任务完成率** | 统计完成的学习任务比例 |
| **内容浏览深度** | 统计浏览的页面数和时长 |
| **参与度得分** | 参与度的量化评估值 |

#### 参与度得分计算

参与度得分从学习画像数据中获取，是综合多个行为指标计算得出的量化值。

#### 六维雷达图计算

系统计算六维画像数据（用于学习画像展示），其中"学习参与度"维度对应三维度参与度模型中的行为参与度：

| 雷达图维度 | 计算方式 | 满分阈值 | 对应三维度模型 |
| :--- | :--- | :--- | :--- |
| **知识掌握** | 知识点得分的平均值 | 100 | 认知参与度 |
| **学习参与度** | 参与度量化值 | 100 | 行为参与度 |
| **学习目标** | 根据学习目标数量计算 | 100 | 情感参与度 |
| **学习偏好** | 已填写的偏好项比例 | 100 | 行为参与度 |
| **概念清晰度** | 根据错误概念数量计算 | 100 | 认知参与度 |
| **画像置信度** | 置信度得分的平均值 | 100 | 系统评估 |

#### 三维度参与度模型实现

| 三维度模型维度 | 数据来源 | 量化方式 |
| :--- | :--- | :--- |
| **认知参与度** | 知识点掌握程度、概念清晰度 | 知识掌握得分 + 概念清晰度得分 |
| **情感参与度** | engagement.score、学习目标 | engagement.score + 学习目标得分 |
| **行为参与度** | 学习时长、互动频率、任务完成率 | 综合行为数据统计 |

#### 参与度等级判断

| 等级 | 判断方式 | 表现 |
| :--- | :--- | :--- |
| **高参与** | engagement.score较高且稳定 | 主动学习、频繁互动 |
| **中参与** | engagement.score中等 | 按时学习、适度互动 |
| **低参与** | engagement.score较低或波动大 | 学习时间少、互动少 |
| **不参与** | engagement.score为0或无记录 | 长期未学习 |

#### 学习评估中的参与度

在_build_learning_assessment_snapshot函数中，参与度数据用于综合评估：

| 评估维度 | 计算方式 | 权重 |
| :--- | :--- | :--- |
| **学习进度** | 课程完成数和进度百分比 | 30% |
| **练习测试** | 近7天练习次数和平均得分 | 40% |
| **资源使用** | 资源访问和使用情况 | 30% |

## 14. 学习评估

学习评估功能提供多种评估方式，全面检测学习成果。本系统基于形成性评估与总结性评估理论 [6] 和项目反应理论（Item Response Theory, IRT）[7]，实现精准的学习评估。

### 14.1 学习评估页面

系统提供专门的"学习评估与动态调优"页面，集中展示评估结果和系统调整：

#### 页面入口

点击导航栏中的"评估调优"进入评估页面。

#### 页面内容

页面顶部展示关键评估指标：
- **最近7天练习**：近一周触发的练习次数
- **平均得分**：用于观察近期理解稳定度
- **当前薄弱点**：仍需优先补弱的知识点数量
- **已调优路径**：已根据学习状态重排的路径版本
- **自动触发调优**：由小测信号自动重排路径的次数

#### 最近练习与评估证据

左侧面板展示最近几次测验结果：
- 本次得分和正确题数
- 当前难度阶段
- 涉及的知识标签
- 建议回看的资料页面
- 回看资料的快捷入口

#### 最新路径调优结果

右侧面板展示系统根据评估信号进行的路径调整：
- 当前优先学习阶段
- 调优焦点说明
- 调优类型和触发来源
- 薄弱点数量和进度信号
- 查看路径详情和重排入口

#### 当前薄弱点焦点

展示当前需要优先处理的知识点：
- 知识点名称和来源资料
- 错误次数和连续错误次数
- 回看资料和AI辅导入口

### 14.2 评估类型

系统支持多种评估方式 [6]：

| 评估类型 | 描述 | 特点 |
| :--- | :--- | :--- |
| **形成性评估** | 在学习过程中进行的评估 | 及时反馈、针对性强 |
| **总结性评估** | 在学习结束后进行的评估 | 全面检测、综合性强 |
| **自我评估** | 学生自行进行的评估 | 反思学习、自我认知 |

### 14.3 评估方法

系统采用多种评估方法：

| 方法 | 描述 | 适用场景 |
| :--- | :--- | :--- |
| **选择题** | 从多个选项中选择正确答案 | 基础知识检测 |
| **填空题** | 填写正确的答案 | 概念理解检测 |
| **简答题** | 用简短文字回答问题 | 理解和应用能力检测 |

### 14.4 评估反馈

| 反馈类型 | 描述 |
| :--- | :--- |
| **即时反馈** | 答题后立即显示正确答案和解析 |
| **详细解析** | 提供题目涉及知识点的详细讲解 |
| **错误分析** | 分析错误原因，指出问题所在 |
| **改进建议** | 根据错误提供针对性的学习建议 |
| **成绩统计** | 统计各题型的正确率和得分情况 |

### 14.5 评估报告

系统生成综合评估报告：

- **成绩概览**：各项评估的得分情况
- **知识点分析**：各知识点的掌握程度评估
- **能力分析**：各项能力的综合评估
- **进步趋势**：多次评估的成绩变化趋势
- **学习建议**：基于评估结果的改进建议

### 14.6 项目反应理论（IRT）

系统采用项目反应理论 [7] 进行测验设计：

| 参数 | 描述 | 作用 |
| :--- | :--- | :--- |
| **题目难度** | 衡量题目的难易程度 | 调整题目难度，匹配学生水平 |
| **题目区分度** | 衡量题目区分不同能力学生的能力 | 筛选有效题目 |
| **猜测参数** | 衡量学生随机猜测答对的概率 | 纠正猜测对成绩的影响 |
| **学生能力** | 衡量学生的知识掌握水平 | 评估学习效果 |

### 14.7 实现方式

学习评估系统采用多维度数据收集和智能分析的方式实现。

#### 数据收集流程

系统收集以下类型的数据：

| 数据类型 | 收集方式 |
| :--- | :--- |
| **测验成绩** | 用户完成练习题后记录 |
| **知识点统计** | 统计每个知识点的答题情况 |
| **错误模式** | 记录错误类型和连续错误次数 |
| **薄弱点归档** | 阶段性完成后归档薄弱点记录 |

#### 路径调优机制

| 调优类型 | 触发条件 | 调优方式 |
| :--- | :--- | :--- |
| **资料小测触发** | 用户完成资料练习后 | 根据答题结果调整路径 |
| **手动重排触发** | 用户主动点击重排按钮 | 重新生成学习路径 |
| **薄弱点重排触发** | 薄弱点连续错误达到阈值 | 优先安排薄弱点学习 |
| **评估信号触发** | 系统自动评估后 | 基于综合分析调整路径 |

#### 评估指标计算

| 指标 | 计算方式 |
| :--- | :--- |
| **最近7天练习** | 统计近7天的练习次数 |
| **平均得分** | 计算近7天所有练习的平均分 |
| **当前薄弱点** | 统计存在错误的知识点数量 |
| **已调优路径** | 获取学习计划中的调优记录 |

## 15. 资源推送中心

资源推送中心集中展示系统根据学习路径、当前薄弱点和课程资料为您推荐的资源，帮助您高效学习。

### 15.6 实现方式

资源推送中心采用基于学习路径和薄弱点的智能推荐方式，综合考虑学习状态和知识需求进行资源匹配。

#### 推荐算法

系统从学习路径中提取模块和课程信息进行匹配：

| 推荐维度 | 分析方式 |
| :--- | :--- |
| **路径阶段匹配** | 分析学习路径中的阶段信息 |
| **薄弱点命中** | 匹配薄弱点与学习任务的标题和目标 |
| **学习状态** | 分析最近的学习进度和表现 |

#### 资源卡片生成流程

| 步骤 | 描述 |
| :--- | :--- |
| **1** | 获取最新学习计划 |
| **2** | 匹配相关的课程和资料 |
| **3** | 遍历学习模块和课程任务 |
| **4** | 生成行动建议 |
| **5** | 匹配薄弱点与课程内容 |
| **6** | 构建推荐理由 |

#### 资源卡片内容

| 内容类别 | 描述 | 示例值 |
| :--- | :--- | :--- |

| **行动建议** | 推荐的操作列表 | 进入学习、开始练习等 |
| **资料匹配信息** | 资料片段的匹配信息 | 匹配的章节或页面 |
| **优先级** | 资源的优先级排序 | 根据学习阶段和任务顺序 |

#### 推荐理由生成逻辑

## 18. 常见问题

### 18.1 登录问题

**Q：忘记密码怎么办？**
A：在登录页面点击"忘记密码"链接，输入注册邮箱，查收重置链接。

**Q：登录失败提示用户名或密码错误？**
A：请检查用户名和密码是否正确，注意区分大小写。

### 18.2 AI功能问题

**Q：AI生成课程失败？**
A：请检查讯飞星火API凭证是否配置正确，网络连接是否正常。

**Q：AI回答质量不佳？**
A：可以尝试重新构建学习画像，提供更详细的学习信息。

### 18.3 课程生成问题

**Q：课程生成失败？**
A：请检查讯飞星火API凭证是否配置正确，网络连接是否正常。

**Q：课程生成时间过长？**
A：课程生成时间取决于主题内容和生成选项，建议耐心等待。

**Q：生成的课程内容不符合预期？**
A：可以尝试在"学习目标与基础"文本框中提供更详细的描述，或重新构建学习画像。

### 18.4 PPT导出问题

**Q：PPT导出失败？**
A：请检查课程内容是否完整，网络连接是否正常。

**Q：PPT中代码显示不全？**
A：代码块会自动换行，建议在PPT中使用合适的字体大小。

### 18.5 知识追踪问题

**Q：知识追踪结果不准确？**
A：知识追踪需要一定的学习数据积累，请完成更多学习任务后再查看。

**Q：如何查看知识点的详细分析？**
A：在知识追踪页面点击具体知识点，查看详细分析和学习建议。

### 18.6 学习路径问题

**Q：学习路径不符合我的需求？**
A：可以手动调整学习路径，或重新构建学习画像。

**Q：学习路径更新不及时？**
A：学习路径会根据学习进度自动更新，也可以手动触发更新。

### 18.7 情感识别问题

**Q：情感识别结果不准确？**
A：情感识别基于多种因素分析，单次结果可能有偏差，建议查看趋势分析。

**Q：如何关闭情感识别功能？**
A：在个人设置中可以关闭情感识别相关功能。

### 18.8 参与度问题

**Q：参与度评分偏低？**
A：可以通过增加学习时长、完成更多任务、积极互动来提高参与度。

**Q：参与度数据更新不及时？**
A：参与度数据通常在学习活动结束后自动更新，最长延迟不超过24小时。

### 18.9 评估问题

**Q：测验题目难度过高或过低？**
A：题目难度基于学习画像生成，可以通过更新画像调整难度。

**Q：评估结果如何影响学习路径？**
A：评估结果会反馈到知识追踪系统，影响后续学习路径的规划。

### 18.10 技术支持

如果遇到其他问题，请联系技术支持：
- 查看系统日志获取详细错误信息
- 检查网络连接和服务器状态
- 确认依赖包版本是否符合要求
